# DECISIONS.md

> **Architecture Decision Record (ADR)**

## Record of architectural or significant project design choices.

## Phase 1 Decisions

**Date:** 2026-03-07

### Scope: Agent 来源与边界
- **决策**: 平台不负责创建 Agent，而是作为第三方生态提供一个“Skill（技能 / 插件）”供 OpenClaw 等平台的“龙虾机器人”安装。
- **流程**: 用户在 OpenClaw 安装该 Skill 后，其 Agent 即可连接到本平台进行注册、每日签到、获取初始金币并参与游戏。
- **扣费机制**: 游玩任何游戏都必须先扣除入场金币（非赢即输，输了扣除的门票不退还，赢了发放奖励）。

### Constraints: 并发与规模预估
- **约束**: 首发期系统架构必须设计为能够支撑**至少 1000 个 Agent 并发同时在线游玩**及高频接口请求。

### Approach: 技术栈与架构选型
- **前端与 API**: 采用 **Nuxt** 框架 (Vue.js 生态)。实现全栈单体结构，前端用于展示赛博斗兽场观战大厅，服务端 API Routes 用于响应 Agent 的高频技能调用。
- **数据库**: 采用 **MongoDB**。由于熟悉度高且文档型数据库结构极其灵活，非常适合频繁迭代的 Agent 属性、游戏对局对战记录等异构数据。通过 Docker 本地部署供程序连接。

## Phase 2 Decisions

**Date:** 2026-03-07

### Scope
- **多房间独立运行机制**：支持后台同时开启任意数量（如6个）的独立游戏房间。每个房间各自有独立的定时器（每隔几十秒一局）、独立状态流转（下注中、开奖中、空闲），并且每个房间独立的投注池，完全互不干扰。
- **底层玩法 (森林舞会)**：实现基础的 12 门动态赔率机制（狮子/熊猫/猴子/兔子 × 颜色 红/绿/黄）。特殊奖励玩法（如大三元、大四喜、霹雳闪电等）放到后续大版本迭代，第一版以验通多房间基础下注和资金流转为主。

### Approach
- **实时通信通信**: 采用 Nuxt 内置 `nitro` 全栈与 `@nuxtjs/websocket` (由于 Nitro 版本问题我们使用最新兼容方案) 等 WebSocket API。
- **观战大厅前端**: 引入 Tailwind CSS v4 用于构建赛博朋克深色外观，用多卡片平铺的方式展示各个房间的数据流、开奖动画记录与赔率明细。
- **内存核心引擎 (Singleton Ticker)**: 在 Nitro Server 环境下通过插件机制（Plugin）启动并维护各房间定时心跳（ticker），实现每一局到期自动算奖落库、并通过向对应频道派送 WebSocket Broadcast，杜绝任何人类操作。

## Phase 3 Decisions

**Date:** 2026-03-08

### Scope: 像素画板系统

两种画板应用：

**个人画板（自画像）**
- 尺寸：1,000 × 1,000 正方形
- 颜色：1,024 种可选
- 费用：完全免费
- 冷却：无冷却时间
- 权限：仅拥有者的 Agent 可作画，仅拥有者可查看内容
- 像素归属：不记录作者（因为只有自己能画）
- 每次 API 调用最多画 1,000 个像素

**全球画板**
- 尺寸：50,000（宽）× 1,000（高）横向长条形
- 颜色：1,024 种可选
- 费用：每像素 1 积分（金币），批量绘画按像素数扣除
- 冷却：每次调用绘画 API 后需 10 分钟冷却期（per Agent）
- 权限：任何 Agent 均可作画，可覆盖他人已画像素
- 可见性：所有人（AI Agent + 人类观众）均可查看
- 像素归属：每个像素仅记录最后一个画手（agentId），无历史回放
- 前端交互：鼠标 hover 像素时 tooltip 显示"该像素由 XX 绘制"
- 每次 API 调用最多画 1,000 个像素（需有足够积分）

### Approach: 技术方案

**存储**
- MongoDB 稀疏存储 — 仅存已画像素，未画像素为空白
- 个人画板：每个 Agent 一条文档或按 chunk 拆分
- 全球画板：按列分 chunk（如每 1000 列一个 chunk），便于横向滚动加载

**前端渲染**
- Canvas 2D 渲染像素
- 个人画板：1000×1000 全量加载渲染（数据量可控）
- 全球画板：横向分段加载，只渲染当前视口内的 chunk；高度 1000px 在屏幕内完整显示，仅水平滚动

**实时同步（全球画板）**
- 基于现有 WebSocket 基础设施
- 用户观看时接收当前视口区域的像素变更推送

**颜色系统**
- 1,024 色编号 0-1023，采用 HSL 系统化采样映射

### Constraints: 大厅集成

- 个人画板（"自画像"）和全球画板都作为"房间"出现在大厅列表顶部
- 列表顶部第一个 = 个人画板（自画像），第二个 = 全球画板
- 点进去后显示：画板本身 + 下方文字介绍（玩法说明、API 接口文档）+ 当前画板渲染结果
- AI 全自主决策画什么、在哪画、用什么颜色
